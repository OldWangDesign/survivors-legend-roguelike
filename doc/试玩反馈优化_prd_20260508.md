# 试玩反馈优化需求文档

> 文档状态：已确认方向，待开发拆分
> 适用项目：幸存者传说（Survivor Legend）
> 目标：基于玩家试玩反馈，优先优化主角识别与表现力、技能展示、Boss 战体验、后期压力和地图玩法
> 说明：手机专项性能优化本轮暂不纳入范围

---

## 1. 背景

玩家试玩后提出 6 条核心反馈：

1. 主人物设计大一点。
2. 各种技能多了之后特别乱，展示可以优化。
3. 大 Boss 不经验。
4. 怪太多，打到后面画面特别乱。
5. 好玩性可以再加强，比如地图、关卡设置。
6. 手机玩太卡怎么改善体验。

经讨论后，本轮不再把"手机玩太卡"作为产品需求重点，因为该问题已单独优化。其余 5 点将作为本轮试玩体验优化的核心范围。

进一步讨论中发现，"大 Boss 不经验"不仅是击杀奖励感知弱的问题，Boss 的外观、登场、技能表现和阶段转换也缺乏足够的冲击力，导致整体 Boss 战体验不够惊艳。因此本轮在 Boss 奖励反馈之外，新增 Boss 战体验强化需求。

同时，Boss 视觉大幅强化后，主角需要保持足够的"主角感"。当前 10 个角色的被动技能完全没有视觉反馈，既影响角色辨识度，也让玩家无法直观读懂自己的状态。因此新增主角被动视觉反馈和状态表现需求。

---

## 2. 总体目标

本轮优化的目标不是重做游戏系统，而是解决试玩中最直接影响体验的问题：

- 玩家能更容易看清主角。
- **主角有"主角感"：被动技能激活时有视觉反馈，升级有闪光，低血量有危险提示。**
- 后期技能和怪物不再过度遮挡画面。
- Boss 击杀后有明确奖励感。
- **Boss 战全流程（登场→战斗→狂暴→击杀）都有强烈的存在感和仪式感。**
- 后期难度从单纯怪海转向机制压力。
- 地图和关卡有更多局内变化。

## 5. 详细需求

### 5.1 主角识别优化

#### 需求描述

后期画面中敌人、技能、经验球和特效较多，玩家需要更容易定位主角。优化应优先提升可读性，而不是改变玩法难度。

#### 规则

- 主角视觉尺寸默认放大到 `1.25x`。
- 如试玩仍反馈不明显，可上调到 `1.35x`。
- 不改变主角碰撞区、受击范围、拾取范围。
- 增加脚底识别环，**常驻显示**，使用半透明浅蓝色（低存在感但一眼可辨），不随战斗状态隐藏。

#### 验收标准

- [ ] 主角视觉明显大于当前版本。
- [ ] 主角受击范围不随视觉放大而变大。
- [ ] 脚底识别环在后期怪多时仍清晰可见。
- [ ] 后期怪多时，玩家仍能快速判断自己位置。

---

### 5.2 主角被动视觉与状态表现

#### 需求描述

游戏有 10 个可选角色，每个角色有独特的被动技能。当前被动激活时没有任何视觉反馈，玩家无法直观读懂自己的状态（如嗜血是否叠满、超频是否激活、神圣庇护是否就绪）。同时 Boss 视觉大幅强化后，主角需要通过被动视觉反馈保持"主角感"，不会在画面中显得平淡。

主角的视觉设计原则：**辨识度 > 华丽度**。不做大面积粒子和华丽光环（与"降噪"矛盾），而是用**紧贴身体的小范围光效**传达状态信息。

#### 5.2.1 被动技能视觉反馈（10 个角色）

每个角色在被动激活/持续生效时，身体周围显示对应的小范围视觉效果。使用 `_draw()` 代码绘制，不需要美术资源。

| # | 角色 | 被动名称 | 被动机制 | 视觉效果（激活态） | 颜色 | 绘制方式 |
|---|---|---|---|---|---|---|
| 1 | 神秘幸存者 | 求生本能 | 血量 < 30% 时移速 +25% | 身体边缘**白色风速线**：2-3 条短弧线沿移动方向拖出 | 白色 alpha 0.5 | `draw_arc` 动态偏移，仅低血量时绘制 |
| 2 | 蓝甲骑士 | 铁壁 | 受伤减少 20%（常驻） | 身体外围**淡蓝色六边形护盾轮廓**，受击瞬间闪亮 0.15s | 淡蓝 alpha 0.25，闪亮时 0.6 | `draw_arc` 6 段拼接，受击时 `_shield_flash_timer` 提升 alpha |
| 3 | PICO-8 复古 | 复古幸运 | 经验值 +20%（常驻） | 拾取经验球时，经验球飞入身体的轨迹变为**金色闪光轨迹** | 金色 | 在经验球拾取逻辑中加金色 `spark_burst`（2-3 颗） |
| 4 | 翠林游侠 | 鹰眼 | 投射物射程和速度 +25%（常驻） | 面朝方向有一个**淡绿色小准星图标**，跟随朝向旋转 | 浅绿 alpha 0.3 | `draw_line` 十字准星，距身体 `hit_radius * 2` |

#### 5.2.2 通用状态反馈（所有角色共享）

| 状态 | 视觉效果 | 说明 |
|---|---|---|
| **升级** | 全身短暂金色发光 0.3s + 向外扩散金色冲击波 | 利用已有的 `ring_wave`，颜色改为金色 |
| **低血量（< 30%）** | 身体边缘**红色脉动呼吸光**，频率随血量降低而加快 | `_draw()` 中画一层红色半透明圆，alpha 用 `sin()` 驱动 |
| **无敌帧** | 保持当前实现（闪白 + 半透明闪烁） | 不做改动 |

#### 设计原则

- 所有被动视觉效果**紧贴身体**，范围不超过 `hit_radius * 2`，避免和武器特效混淆。
- 常驻被动（铁壁、鹰眼、复古幸运、生命之泉）的视觉低调（alpha 0.2-0.3），不干扰战斗。
- 触发型被动（嗜血叠层、魔力涌动释放、神圣庇护触发、求生本能激活）的视觉明显（alpha 0.5-0.8），让玩家知道"发生了什么"。
- 被动视觉与脚底识别环不冲突：识别环在最底层，被动效果在身体层。

#### 验收标准

- [ ] 10 个角色在被动激活时各有独特的视觉反馈。
- [ ] 赤焰武士嗜血 3 层时视觉明显区别于 1 层。
- [ ] 暗夜法师魔力涌动最后 3 秒有明确的蓄力感。
- [ ] 公主婚礼能一眼看出神圣庇护是否就绪。
- [ ] 赛博朋克超频激活与未激活在视觉上有清晰区分。
- [ ] 升级瞬间全身金色闪光可被注意到。
- [ ] 低血量红色脉动不会与受伤闪白混淆。
- [ ] 所有被动视觉效果不超出身体范围过远，不增加画面噪音。

### 5.3 技能展示降噪

#### 需求描述

技能变多后，画面中的伤害数字、拖尾、粒子和普通命中特效会相互叠加，影响玩家判断危险、敌人和自身位置。

#### 规则

- 伤害数字限流：同屏上限 `40-50` 个，每秒生成上限 `50-60` 条。**暴击数字不受限流影响**，保证大伤害反馈。
- 普通武器投射物拖尾长度缩短。**进化武器保留完整拖尾**，因为拖尾是进化后"变强"的视觉反馈。
- 普通命中特效、装饰粒子可降低表现强度。
- 大招、进化技能、Boss 技能仍保留强表现。
- 视觉优先级从高到低为：
  1. 主角位置。
  2. 敌人危险攻击和 Boss 技能。
  3. Boss 奖励、宝箱、大经验球。
  4. 普通敌人和经验球。
  5. 小伤害数字、普通命中特效、装饰粒子。

#### 验收标准

- [ ] 后期不会因为伤害数字过多遮挡战斗。
- [ ] 暴击数字仍然能正常显示，不被限流吃掉。
- [ ] 进化武器拖尾与普通武器有明显差异。
- [ ] 玩家能区分 Boss 技能和普通技能特效。
- [ ] 技能仍保留割草爽感，不显得过于朴素。

---

### 5.4 Boss 奖励反馈

#### 需求描述

玩家反馈"大 Boss 不经验"，本质是 Boss 击杀后的奖励感知不足。本轮需要让 Boss 死亡瞬间产生明确奖励反馈。

#### 规则

- Boss 死亡后掉落大量经验球。
- 不做经验自动吸入。
- 掉落 `8` 个大经验球，Boss 总经验均分到 8 个球，**单颗保底不低于 20 xp**（确保达到大经验球视觉阈值）。
- 经验球围绕 Boss 死亡点小范围环形散落，散落半径约 `30-40` 像素。
- Boss **100% 必掉宝箱**，稀有度为 `60%` 史诗、`40%` 稀有，不掉普通箱。
- 显示明确提示：使用 **HUD 中央横幅**（类似怪潮预警样式）显示"Boss 掉落大量经验！"，持续约 `2` 秒。

#### 验收标准

- [ ] Boss 死亡后玩家能立即看到经验球爆发。
- [ ] 每颗经验球视觉为大经验球尺寸。
- [ ] Boss 经验球不会过于分散，玩家能顺路拾取。
- [ ] Boss 死亡后必定出现宝箱，且不会掉普通箱。
- [ ] HUD 中央横幅提示清晰可见，不被飘字遮挡。

### 5.5 Boss 战体验强化

#### 需求描述

当前 3 个 Boss（骸骨领主、暗影巫妖、血月魔王）共用同一个 sprite，体型差异小（28/32/36），光环相同，登场只有简单的 lerp 走入，技能视觉和普通怪差不多，50% 狂暴转换几乎不被注意。需要从外观、登场、技能表现和阶段转换四个维度全面提升 Boss 战的冲击力。

#### 5.5.1 Boss 体型拉开（P0）

当前 Boss 体型太接近普通怪，在怪群中不够突出。

| Boss | 当前 size | 新 size | 对比普通骷髅（12） |
|---|---|---|---|
| 骸骨领主 | 28 | **40** | 3.3x |
| 暗影巫妖 | 32 | **48** | 4.0x |
| 血月魔王 | 36 | **56** | 4.7x |

规则：
- 体型增大后，光环半径和技能范围按比例调整，保持视觉协调。
- Boss 碰撞体积同步增大（Boss 本身就应该占空间）。

#### 5.5.2 登场演出强化（P0）

替换当前的 2 秒 lerp 走入，改为有仪式感的登场序列。

| 时间点 | 事件 | 说明 |
|---|---|---|
| 0.0s | 屏幕边缘红色脉冲预警 | 已有，保留 |
| 0.0s | 小怪停止生成 | 已有，保留 |
| 1.5s | 屏幕短暂变暗 | alpha 0.3 暗色遮罩，0.3s 淡入 |
| 2.0s | Boss 名称大字居中显示 | 如"骸骨领主"，带缩放弹入动画，持续 1.2s |
| 2.0s | Boss 从屏幕外走入 | 同时全身发光描边效果 |
| 3.2s | 名称淡出，遮罩淡出 | 过渡到正常画面 |
| 3.2s | Boss 到达目标位置，释放冲击波 | ring_wave，正式开战 |

规则：
- 总登场时间约 `3.2` 秒（当前 2 秒，增加 1.2 秒）。
- 登场期间 Boss 无敌，不可被攻击。
- 登场动画期间游戏不暂停，玩家仍可移动（为走位留时间）。

#### 5.5.3 阶段转换演出（P0）

当前 50% 血量狂暴只有加速和红色脉冲，太安静。

| 时间点 | 事件 | 说明 |
|---|---|---|
| 触发瞬间 | Boss 暂停移动 0.5s | "蓄力"感，让玩家注意到变化 |
| 0.0s | 全屏红闪 | 比当前更强的 screen_flash |
| 0.0s | Boss 短暂放大到 `1.3x` | 0.3s 后弹回原尺寸，视觉冲击 |
| 0.2s | 大冲击波 | ring_wave 半径 = size * 8，有击退感 |
| 0.3s | HUD 中央横幅"Boss 狂暴！" | 持续 1.5s，红色文字 |
| 0.5s | Boss 恢复移动 | 正式进入狂暴状态 |

规则：
- 总暂停仅 `0.5` 秒，不影响战斗节奏。
- 冲击波不造成伤害，仅做视觉表现。

#### 5.5.4 Boss 专属光环（P1）

当前 3 个 Boss 光环完全相同（脉冲半透明圆），缺乏个性。使用 `_draw()` 代码绘制差异化光环，不需要美术资源。

| Boss | 光环风格 | 描述 |
|---|---|---|
| 骸骨领主 | 骨白色 + 旋转骨片 | 3-4 个小三角/菱形图形绕 Boss 缓慢旋转，偶尔闪烁 |
| 暗影巫妖 | 紫色暗能量漩涡 | 紫色半透明螺旋线持续旋转，有内外两层 |
| 血月魔王 | 红色血焰脉动 | 不规则红色火焰状边缘，强烈脉动呼吸感 |

规则：
- 光环在 Boss 存活期间持续绘制。
- 狂暴后光环强度增大（透明度提升、旋转加速或范围扩大）。

#### 5.5.5 技能预警和打击感强化（P1）

当前 Boss 技能的视觉反馈和普通怪攻击差不多，缺乏威胁感。优先增强感知最明显的几个技能。

| 技能 | 当前表现 | 增强方案 |
|---|---|---|
| 骨刺地裂 | 一条线 | 增加 `0.4s` 地面裂纹预警（红色虚线闪烁），然后骨刺才出来 |
| 死亡冲锋 | tween 移动 | 冲锋路径上留 `3-4` 个残影（渐隐的半透明 Boss 影像） |
| 灵魂弹幕 | 5 个小球 | 弹丸增加短拖尾（3-4 帧），命中时小型爆炸闪光 |
| 血月之怒 | 红闪 + ring_wave | 增加 `1s` 蓄力动画（Boss 缩小再膨胀）+ 地面圆形预警区（安全区 vs 危险区） |
| 黑暗领域 | ring_wave 一闪 | 改为持续可见的紫色半透明区域，存在期间持续显示 |

规则：
- 预警动画不增加额外伤害，只做视觉提示。
- 增强后的技能视觉应明显强于普通怪攻击和玩家技能。

#### 验收标准

- [ ] 三个 Boss 体型在怪群中一眼可辨，互相之间也有明显大小差异。
- [ ] Boss 登场时有名称标题显示和画面变暗效果。
- [ ] 50% 血量狂暴转换时有明显的全屏演出，玩家不会错过。
- [ ] 三个 Boss 的光环风格各不相同。
- [ ] 骨刺地裂有预警、死亡冲锋有残影、血月之怒有蓄力。
- [ ] Boss 战整体感受明显强于普通怪战斗。

### 5.6 后期压力从怪海转向机制

#### 需求描述

后期如果主要靠大量普通怪制造压力，会带来画面混乱和疲劳感。本轮应降低无意义怪海，增加可读、可应对的机制压力。

#### 规则

- 中后期（约 5 分钟后开始生效）普通怪数量先减少约 `20%`（通过降低 spawn 频率或单次 spawn 数量实现）。
- Boss 战期间减少小怪干扰：Boss 存活时，小怪生成速率降低 `50%`，让 Boss 成为视觉和玩法焦点。
- 第一批机制怪为：
  - **冲锋怪**：短暂预警后直线冲刺，考验走位。
  - **远程怪**：保持距离并发射低频弹幕，迫使玩家移动。
- 自爆怪、护盾怪放第二批。

#### 冲锋怪参数

| 属性 | 值 | 说明 |
|---|---|---|
| 血量 | 普通怪的 `1.5-2x` | 不能太脆，需要玩家注意 |
| 移动速度（普通） | 普通怪的 `0.8x` | 平时比普通怪略慢 |
| 冲刺速度 | 普通怪的 `3-4x` | 冲刺时快速逼近 |
| 预警时间 | `0.8-1.0` 秒 | 红色闪烁或蓄力动画 |
| 冲刺距离 | `150-200` 像素 | 直线冲刺，中途不转向 |
| 冲刺伤害 | 普通怪的 `2x` | 被命中代价高 |
| 冲刺冷却 | `3-4` 秒 | 不会连续冲刺 |
| 首次出现时间 | 约 `5` 分钟 | 与怪量收敛同步 |

#### 远程怪参数

| 属性 | 值 | 说明 |
|---|---|---|
| 血量 | 普通怪的 `0.8-1.0x` | 较脆，鼓励玩家主动击杀 |
| 保持距离 | `120-160` 像素 | 始终与玩家保持距离 |
| 弹幕速度 | 中速，可被走位躲避 | 不做追踪弹 |
| 射击间隔 | `2-3` 秒 | 低频，但持续骚扰 |
| 弹幕伤害 | 普通怪碰撞的 `1.0-1.5x` | 单发不致命，累积有压力 |
| 首次出现时间 | 约 `5` 分钟 | 与冲锋怪同批出现 |

#### 验收标准

- [ ] 后期画面比纯怪海更清晰。
- [ ] 冲锋怪有明确的预警动画，玩家能看懂并躲避。
- [ ] 远程怪弹幕可被走位回避，不出现无法回避的情况。
- [ ] 怪物减少后，整体压力不明显下降（机制怪补回挑战）。
- [ ] Boss 战中 Boss 不会被小怪和特效淹没。

### 5.7 地图与关卡变化

#### 需求描述

当前局内变化主要来自怪物数量和数值增长，玩家路线决策较少。本轮先加入低成本地图事件，提高局内变化。

#### 规则

- 第一批地图事件：
  - **危险区**：出现前有预警，进入后持续受到伤害。
  - **治疗点**：玩家靠近后恢复生命。
- 第二批再考虑资源点。
- 暂不做复杂障碍物、寻路地图或完整随机地图生成。
- 地图事件在游戏 `1.5-2` 分钟后开始出现（前期让玩家熟悉基本战斗）。
- 同屏最多 `1` 个危险区 + `1` 个治疗点，避免地图事件本身造成视觉混乱。

#### 危险区参数

| 属性 | 值 | 说明 |
|---|---|---|
| 预警时间 | `2` 秒 | 出现红色虚线圆圈闪烁 |
| 持续时间 | `6-8` 秒 | 激活后存在时间 |
| 半径 | `80-100` 像素 | 中等范围，不能过大 |
| 每次伤害 | 玩家最大血量的 `3-5%` | 持续伤害，停留越久越危险 |
| 伤害间隔 | `0.5` 秒 | 高频低伤，催促玩家离开 |
| 视觉 | 预警：红色虚线圆；激活：红色半透明区域 + 地面裂纹效果 | 需要与敌人攻击有区分度 |

#### 治疗点参数

| 属性 | 值 | 说明 |
|---|---|---|
| 治疗量 | 玩家最大血量的 `15-20%` | 一次性恢复，有拾取价值 |
| 存在时间 | `8-12` 秒 | 不会一直在，需要抓时机 |
| 触发半径 | `25-30` 像素 | 需要主动靠近 |
| 视觉 | 绿色光圈 + 十字图标 | 明确的治疗暗示 |
| 拾取后效果 | 绿色 +HP 飘字 + 光圈扩散消散 | 正向反馈 |

#### 验收标准

- [ ] 危险区出现前有明确的 2 秒预警动画。
- [ ] 玩家进入危险区会受到持续伤害，离开后伤害立刻停止。
- [ ] 治疗点触发后有明确的恢复反馈。
- [ ] 地图事件不会遮挡主角、Boss 技能或经验球（z-index 低于主角和 Boss）。
- [ ] 同屏地图事件数量不超过上限。

## 6. 建议开发顺序

| 顺序 | 内容 | 说明 |
|---|---|---|
| 1 | 主角视觉放大 | 成本低，感知强，风险小 |
| 2 | Boss 体型拉开 + 登场演出 + 狂暴演出 | P0 三件套，改数值+加演出，让 Boss 立刻"不一样" |
| 3 | Boss 奖励反馈 | 解决玩家明确反馈，提升击杀成就感 |
| 4 | 技能展示降噪 | 改善后期画面可读性 |
| 5 | 主角被动视觉 + 升级闪光 + 低血量脉动 | 降噪完成后再加主角表现，确保不冲突 |
| 6 | Boss 专属光环 + 技能预警强化 | P1 锦上添花，进一步提升 Boss 辨识度 |
| 7 | 后期怪物数量收敛 | 先用数值降低画面噪音 |
| 8 | 冲锋怪 + 远程怪 | 用机制补回挑战 |
| 9 | 危险区 + 治疗点 | 增加局内路线选择 |

## 7. 仍需试玩验证的问题

| 问题 | 验证方式 | 预期判断 |
|---|---|---|
| 主角 `1.25x` 是否足够明显 | 后期 5-10 分钟试玩观察 | 如果仍难找，调到 `1.35x` |
| 被动视觉是否增加画面噪音 | 降噪后测试被动效果 | 如果显杂，降低 alpha 或缩小范围 |

| Boss 新体型（40/48/56）是否合适 | Boss 战期间观察 | 如果太大遮挡操作，适当缩小；如果仍不突出，继续放大 |
| Boss 登场 3.2s 是否太长 | 观察玩家是否不耐烦 | 如果嫌慢可压缩名称显示时间到 0.8s |
| Boss 掉 8 个大经验球是否够爽 | 击杀 Boss 后观察玩家反应 | 如果仍不明显，提高经验球尺寸或增加提示 |
| 普通怪减少 `20%` 是否影响爽感 | 对比调整前后后期压力 | 如果太空，增加机制怪而不是加回普通怪 |
| 冲锋怪和远程怪是否易读 | 观察玩家是否能躲避 | 如果不可读，需要增加预警动作或放慢速度 |
| 危险区和治疗点是否改变路线 | 观察玩家是否主动绕行或争夺 | 如果无感，需要提高奖励或危险强度 |
| Boss 战减少 50% 小怪是否太空 | Boss 战期间观察压力感 | 如果太空可调到 30%，如果仍乱可调到 70% |
