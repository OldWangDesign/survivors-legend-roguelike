## 1. 背景

玩家试玩后提出 6 条核心反馈：

1. 主人物设计大一点。
2. 各种技能多了之后特别乱，展示可以优化。
3. 大 Boss 不经验。
4. 怪太多，打到后面画面特别乱。
5. 好玩性可以再加强，比如地图、关卡设置。
6. 手机玩太卡怎么改善体验。

经讨论后，本轮不再把"手机玩太卡"作为产品需求重点，因为该问题已单独优化。其余 5 点将作为本轮试玩体验优化的核心范围。

进一步讨论中发现，"大 Boss 不经验"不仅是击杀奖励感知弱的问题，Boss 的外观、登场、技能表现和阶段转换也缺乏足够的冲击力，导致整体 Boss 战体验不够惊艳。因此本轮在 Boss 奖励反馈之外，新增 Boss 战体验强化需求。

同时，Boss 视觉大幅强化后，主角需要保持足够的"主角感"。当前 10 个角色的被动技能完全没有视觉反馈，既影响角色辨识度，也让玩家无法直观读懂自己的状态。因此新增主角被动视觉反馈和状态表现需求。

v0.5 进一步识别出：当前游戏有**闯关模式**（关卡 60-180s）和**自由模式**（默认 1200s 或无限）两种，原 PRD 的"5 分钟后触发"等时间型规则在闯关模式下绝大多数情况无法生效。本轮新增模式适配方案。

---

## 2. 总体目标

本轮优化的目标不是重做游戏系统，而是解决试玩中最直接影响体验的问题：

- 玩家能更容易看清主角。
- **主角有"主角感"：被动技能激活时有视觉反馈，升级有闪光，低血量有危险提示。**
- 后期技能和怪物不再过度遮挡画面。
- Boss 击杀后有明确奖励感。
- **Boss 战全流程（登场→战斗→狂暴→击杀）都有强烈的存在感和仪式感。**
- **Boss 有"实体感"：玩家不能直接走进 Boss 身体里，Boss 在画面中占有空间感。**
- 后期难度从单纯怪海转向机制压力。
- 地图和关卡有更多局内变化。
- **闯关模式和自由模式都享受到本轮优化收益。**

## 5. 详细需求

### 5.1 主角识别优化

#### 需求描述

后期画面中敌人、技能、经验球和特效较多，玩家需要更容易定位主角。优化应优先提升可读性，而不是改变玩法难度。

#### 规则

- 主角视觉尺寸默认放大到 `1.25x`。
- 如试玩仍反馈不明显，可上调到 `1.35x`。
- 不改变主角碰撞区、受击范围、拾取范围。
- 增加脚底识别环，**常驻显示**，使用半透明浅蓝色（低存在感但一眼可辨），不随战斗状态隐藏。

#### 实现细节（v0.5 澄清）

- **放大对象**：仅缩放 `player.gd._sprite`（子 Sprite2D 节点），不缩放 `player` 本体。代码中已通过 `GameData.get_player_visual_scale()` 暴露此参数。
- **识别环不跟随放大**：`marker_radius = hit_radius * 1.15`（不再乘 `_visual_scale`）。理由：识别环本意是标定碰撞位置，跟物理一致比跟视觉一致更有用，否则放大后识别环也跟着大反而占画面。
- **碰撞独立性**：当前 `player` 是 `CharacterBody2D` 但 `collision_layer=0, collision_mask=0`，即不参与物理碰撞，所有受击/拾取都用 `hit_radius` 距离判定。视觉缩放对这些数值无影响。

#### 验收标准

- [ ] 主角视觉明显大于当前版本。
- [ ] 主角受击范围不随视觉放大而变大。
- [ ] 脚底识别环在后期怪多时仍清晰可见。
- [ ] 识别环大小与碰撞区一致，不随 `_visual_scale` 改变。
- [ ] 后期怪多时，玩家仍能快速判断自己位置。

### 5.2 主角被动视觉与状态表现

#### 需求描述

游戏有 10 个可选角色，每个角色有独特的被动技能。当前被动激活时没有任何视觉反馈，玩家无法直观读懂自己的状态（如嗜血是否叠满、超频是否激活、神圣庇护是否就绪）。同时 Boss 视觉大幅强化后，主角需要通过被动视觉反馈保持"主角感"，不会在画面中显得平淡。

主角的视觉设计原则：**辨识度 > 华丽度**。不做大面积粒子和华丽光环（与"降噪"矛盾），而是用**紧贴身体的小范围光效**传达状态信息。

#### 5.2.1 被动技能视觉反馈（10 个角色）

每个角色在被动激活/持续生效时，身体周围显示对应的小范围视觉效果。使用 `_draw()` 代码绘制，不需要美术资源。

| # | 角色 | 被动名称 | 被动机制 | 视觉效果（激活态） | 颜色 | 绘制方式 |
|---|---|---|---|---|---|---|

#### 5.2.2 通用状态反馈（所有角色共享）

| 状态 | 视觉效果 | 说明 |
|---|---|---|
| **升级** | 全身短暂金色发光 0.3s + 向外扩散金色冲击波 | `VfxPool.ring_wave(player.global_position, GameData.UI_GOLD, hit_radius * 4.0, 0.4, 3.0)` + `_sprite.modulate` tween 到金色再回白 |
| **低血量（< 30%）** | 身体边缘**红色脉动呼吸光**，频率随血量降低而加快 | `_draw()` 中画一层红色半透明圆，alpha 用 `sin()` 驱动 |
| **无敌帧** | 保持当前实现（闪白 + 半透明闪烁） | 不做改动 |

> **低血量脉动 vs 无敌闪白冲突**：当 `invincible == true` 时**暂时不画**红色脉动（让无敌闪白独占视觉），无敌帧结束后再恢复脉动。优先级：无敌闪白 > 低血量脉动。

#### 5.2.3 设计原则与边界（v0.5 补充）

- 所有被动视觉效果**紧贴身体**，范围不超过 `hit_radius * 2`，避免和武器特效混淆。
- 常驻被动（铁壁、鹰眼、复古幸运、生命之泉）的视觉低调（alpha 0.2-0.3），不干扰战斗。
- 触发型被动（嗜血叠层、魔力涌动释放、神圣庇护触发、求生本能激活）的视觉明显（alpha 0.5-0.8），让玩家知道"发生了什么"。
- 被动视觉与脚底识别环不冲突：识别环在最底层（z=`Z_PLAYER_MARKER`），被动效果在身体层（z=`Z_PLAYER`）。
- **绘制位置约定**：
  - 大部分被动视觉直接在 `player._draw()` 中实现，零额外节点（紧贴身体、性能开销最低）
  - 仅事件型一次性效果用 `VfxPool`：mana_surge 释放冲击波、神圣庇护爆闪、灵魂汲取飞行特效
- **被动视觉不占降噪上限**：所有 `_draw()` 内的被动效果不算入 `float_text_cap` 或 `_get_budget()` 统计。

#### 验收标准

- [ ] 10 个角色在被动激活时各有独特的视觉反馈。
- [ ] 赤焰武士嗜血 3 层时视觉明显区别于 1 层。
- [ ] 暗夜法师魔力涌动最后 3 秒有明确的蓄力感。
- [ ] 公主婚礼能一眼看出神圣庇护是否就绪。

- [ ] 升级瞬间全身金色闪光可被注意到。
- [ ] 低血量红色脉动不会与受伤闪白混淆（无敌期间不画脉动）。
- [ ] Sweetie 回血 "+1" 不会因为 5s 一次的频率把降噪上限吃满。

### 5.3 技能展示降噪

#### 需求描述

技能变多后，画面中的伤害数字、拖尾、粒子和普通命中特效会相互叠加，影响玩家判断危险、敌人和自身位置。

#### 规则

- 伤害数字限流：同屏上限 `40-50` 个，每秒生成上限 `50-60` 条。**暴击数字不受限流影响**（含同屏总数 + 每秒速率双重豁免），保证大伤害反馈。
- 普通武器投射物拖尾长度从默认 `8` 缩短到 `4`。**进化武器保留 `length=8` 完整拖尾**。
- 普通命中特效、装饰粒子按 5.9 清单具体砍除。
- 大招、进化技能、Boss 技能仍保留强表现。
- 视觉优先级从高到低（z-index 实际数值见 5.9）：
  1. 主角位置。
  2. 敌人危险攻击和 Boss 技能。
  3. Boss 奖励、宝箱、大经验球。
  4. 普通敌人和经验球。
  5. 小伤害数字、普通命中特效、装饰粒子。

#### 实现细节（v0.5 澄清）

- **暴击判定阈值**：沿用现有 `weapon_base.gd:79` 的 `dmg >= 20` 阈值，不引入新的 `is_crit` 字段。
- **进化武器判定**：`weapon_type >= GameData.WeaponType.INFERNO_STORM`（枚举里 INFERNO_STORM 之后连续 7 个全是超武，无需查表）。
- **修复一个现有 bug**：`vfx_pool.gd:208-210` 的暴击豁免只覆盖了"每秒速率上限"，没覆盖"同屏总数上限"。需修改：
  ```gdscript
  # 原代码：
  if _float_texts.size() >= cap:
      return false
  # 改为：
  if _float_texts.size() >= cap and not is_crit:
      return false
  ```

#### 验收标准

- [ ] 后期不会因为伤害数字过多遮挡战斗。
- [ ] 暴击数字仍然能正常显示，不被限流吃掉（包括同屏达到 50 个时）。
- [ ] 进化武器拖尾与普通武器有明显差异（视觉上明显更长更密）。
- [ ] 玩家能区分 Boss 技能和普通技能特效。
- [ ] 技能仍保留割草爽感，不显得过于朴素。

### 5.4 Boss 奖励反馈

#### 需求描述

玩家反馈"大 Boss 不经验"，本质是 Boss 击杀后的奖励感知不足。本轮需要让 Boss 死亡瞬间产生明确奖励反馈。

#### 规则

- Boss 死亡后掉落大量经验球。
- 不做经验自动吸入。
- 掉落 `8` 个大经验球，**经验公式：`xp_per_gem = max(20, ceil(boss_xp_value / 8))`，`total = xp_per_gem * 8`**。
- 经验球围绕 Boss 死亡点小范围环形散落，散落半径约 `30-40` 像素。
- Boss **100% 必掉宝箱**，稀有度为 `60%` 史诗、`40%` 稀有，不掉普通箱。
- 显示明确提示：使用 **HUD 中央横幅**（`hud.show_reward_notice` 已存在）显示"Boss 掉落大量经验！"，持续约 `2` 秒。

#### 数值影响说明（v0.5 澄清）

按新公式，**部分早期 Boss 实际经验会上调**：

| Boss xp_value（原值） | 单颗 xp（max(20, ⌈xp/8⌉)） | 新总量 | 是否上调 |
|---|---|---|---|
| 100（关卡 1-5 骸骨领主） | 20 | 160 | ✅ +60% |
| 175（关卡 2-5） | 22 | 176 | ✅ +0.6% |
| 350（关卡 3-5） | 44 | 352 | 几乎不变 |
| 500+ | ≥63 | 500+ | 不变 |

这一上调符合 PRD"爽感强化"意图（早期 Boss 本来就经验偏少），是预期效果。

#### 验收标准

- [ ] Boss 死亡后玩家能立即看到经验球爆发。
- [ ] 每颗经验球视觉为大经验球尺寸（≥20 xp）。
- [ ] Boss 经验球不会过于分散，玩家能顺路拾取。
- [ ] Boss 死亡后必定出现宝箱，且不会掉普通箱。
- [ ] HUD 中央横幅提示清晰可见，不被飘字遮挡。

### 5.5 Boss 战体验强化

#### 需求描述

当前 3 个 Boss（骸骨领主、暗影巫妖、血月魔王）共用同一个 sprite，体型差异小（28/32/36），光环相同，登场只有简单的 lerp 走入，技能视觉和普通怪差不多，50% 狂暴转换几乎不被注意。需要从外观、登场、技能表现、阶段转换、**实体感**五个维度全面提升 Boss 战的冲击力。

#### 5.5.1 Boss 体型拉开（P0）

当前 Boss 体型太接近普通怪，在怪群中不够突出。

| Boss | 当前 size | 新 size | 对比普通骷髅（12） |
|---|---|---|---|
| 骸骨领主 | 28 | **40** | 3.3x |
| 暗影巫妖 | 32 | **48** | 4.0x |
| 血月魔王 | 36 | **56** | 4.7x |

规则：
- 体型增大后，光环半径和技能范围按比例调整，保持视觉协调。
- Boss `enemy_size` 同步增大（影响距离判定的"占地"半径，与软推开机制配合）。

#### 5.5.2 登场演出强化（P0）

替换当前的 2 秒 lerp 走入，改为有仪式感的登场序列。

| 时间点 | 事件 | 说明 |
|---|---|---|
| 0.0s | 屏幕边缘红色脉冲预警 | 已有，保留 |
| 0.0s | 小怪停止生成 | 已有，保留 |
| 1.5s | 屏幕短暂变暗 | alpha 0.3 暗色遮罩，0.3s 淡入 |
| 2.0s | Boss 名称大字居中显示 | 如"骸骨领主"，带缩放弹入动画，持续 1.2s |
| 2.0s | Boss 从屏幕外走入 | 同时全身发光描边效果 |
| 3.2s | 名称淡出，遮罩淡出 | 过渡到正常画面 |
| 3.2s | Boss 到达目标位置，释放冲击波 | ring_wave，正式开战 |

规则：
- 总登场时间约 `3.2` 秒（当前 2 秒，增加 1.2 秒）。
- **登场期间 Boss `take_damage` early return**（不可被攻击）。
- **登场期间 Boss 不造成接触伤害**（所有 `_skill_*` 在 `_entering` 时跳过）。
- 登场动画期间游戏不暂停，玩家仍可移动（为走位留时间）。
- **物理冲突说明**：Boss 是 `extends Node2D` 无物理碰撞体，登场期间玩家可视觉穿过 Boss 走入路径，无需特殊处理。登场结束后由 5.5.4 的软推开机制接管。

#### 5.5.3 阶段转换演出（P0）

当前 50% 血量狂暴只有加速和红色脉冲，太安静。

| 时间点 | 事件 | 说明 |
|---|---|---|
| 触发瞬间 | Boss 暂停移动 0.5s | "蓄力"感，让玩家注意到变化 |
| 0.0s | 全屏红闪 | 比当前更强的 screen_flash |
| 0.0s | Boss 短暂放大到 `1.3x` | 0.3s 后弹回原尺寸，仅缩放子 sprite，不改 hit_radius |
| 0.2s | 大冲击波 | ring_wave 半径 = size * 8，有击退感 |
| 0.3s | HUD 中央横幅"Boss 狂暴！" | 持续 1.5s，红色文字 |
| 0.5s | Boss 恢复移动 | 正式进入狂暴状态 |

规则：
- 总暂停仅 `0.5` 秒，不影响战斗节奏。
- 冲击波不造成伤害，仅做视觉表现。

#### 5.5.4 Boss 软推开机制（P0，v0.5 新增）

当前 Boss 是 `extends Node2D` 无物理碰撞体，所有伤害靠距离判定。这种架构性能优秀（适配 200-300 敌人 + SpatialGrid），但导致 Boss 缺乏"实体感"——玩家可以直接走进 Boss 身体里，与"56px 巨型 Boss 一眼可辨"的视觉宣传矛盾。

**实现**：在 `boss_enemy.gd._physics_process` 中增加约 5 行：

```gdscript
# 软推开：玩家不能站进 Boss 身体里
if is_instance_valid(player) and not _entering and not _dying:
    var to_player := player.global_position - global_position
    var min_dist := enemy_size + player.hit_radius
    var d := to_player.length()
    if d < min_dist and d > 0.01:
        var push := to_player.normalized() * (min_dist - d)
        player.global_position += push
```

**配套**：让 `boss_enemy.gd:371` 的 `apply_knockback` 真正生效（当前是 `pass`），允许玩家武器击退 Boss（击退量取武器配置）。

规则：
- **仅 Boss 加软推开**，普通敌人不加（保持割草手感）。
- 登场期 / 死亡期不生效（避免演出冲突）。
- 物理引擎完全未参与，性能开销 ≈ 一次距离比较。
- 软推开后玩家位置直接修改，不触发 `move_and_slide` 的物理回调。

### 5.6 后期压力从怪海转向机制

#### 需求描述

后期如果主要靠大量普通怪制造压力，会带来画面混乱和疲劳感。本轮应降低无意义怪海，增加可读、可应对的机制压力。

#### 规则

- **自由模式**：约 5 分钟后开始生效，普通怪数量先减少约 `20%`（通过降低 spawn 频率或单次 spawn 数量实现）。
- **闯关模式**：不削减普通怪量（关卡短，本来就需要密度感）。
- Boss 战期间减少小怪干扰：Boss 存活时，小怪生成速率降低 `50%`，让 Boss 成为视觉和玩法焦点。两种模式都生效。
- 第一批机制怪为：
  - **冲锋怪**：短暂预警后直线冲刺，考验走位。
  - **远程怪**：保持距离并发射低频弹幕，迫使玩家移动。
- 自爆怪、护盾怪放第二批。
- **机制怪在两种模式下的接入规则不同**，详见 5.10。

#### 普通怪基准（v0.5 澄清）

冲锋怪 / 远程怪参数表里"普通怪的 X 倍"统一以 **`zombie` (hp=30, size=14, damage=8)** 为基准。

理由：zombie 是当前 ENEMY_TYPES 中体型大、血厚的"标准小怪"，符合机制怪应有的视觉重量。如果用 skeleton 或 bat 当基准，机制怪会过于脆弱，与"考验走位"的设计意图不符。

#### 冲锋怪参数

| 属性 | 值 | 说明 |
|---|---|---|
| 血量 | 50 (zombie 1.7x) | 不能太脆，需要玩家注意 |
| 移动速度（普通） | 24 (zombie 0.8x) | 平时比普通怪略慢 |
| 冲刺速度 | 90-120 (zombie 3-4x) | 冲刺时快速逼近 |
| 预警时间 | `0.8-1.0` 秒 | 红色闪烁或蓄力动画 |
| 冲刺距离 | `150-200` 像素 | 直线冲刺，中途不转向 |
| 冲刺伤害 | 16 (zombie 2x) | 被命中代价高 |
| 冲刺冷却 | `3-4` 秒 | 不会连续冲刺 |
| 首次出现时间 | 自由模式 `5` 分钟；闯关模式按 5.10 接入 | — |

#### 远程怪参数

| 属性 | 值 | 说明 |
|---|---|---|
| 血量 | 24 (zombie 0.8x) | 较脆，鼓励玩家主动击杀 |
| 保持距离 | `120-160` 像素 | 始终与玩家保持距离 |
| 弹幕速度 | 中速，可被走位躲避 | 不做追踪弹 |

| 弹幕伤害 | 8-12 (zombie 1.0-1.5x) | 单发不致命，累积有压力 |
| 首次出现时间 | 同冲锋怪 | — |

#### 机制怪行为细则（v0.5 新增）

- **冲锋怪扑空后**：到达冲刺终点后切回"普通追击"模式，下次 CD 到了再冲刺。**不重复连冲**。
- **远程怪被追近**（玩家距离 < 80px）：立即后撤，**后撤速度 = zombie × 1.2**（即 36/s，平时是 24/s）。撤到保持距离区间（120-160px）后停下继续射击。如果一直被追，持续后撤直到撞到屏幕外被销毁。
- **机制怪是否计入"普通怪 -20%" 统计**：**不计入**。-20% 只针对 bat/skeleton/zombie/ghost 这 4 种。

#### 验收标准

- [ ] 自由模式 5min 后画面比纯怪海更清晰；闯关模式密度保持。
- [ ] 冲锋怪有明确的预警动画，玩家能看懂并躲避。
- [ ] 冲锋怪扑空后切回普通模式，不立即追加冲刺。
- [ ] 远程怪弹幕可被走位回避，不出现无法回避的情况。
- [ ] 玩家追近远程怪时，远程怪会后撤，不发生无脑被秒。
- [ ] 怪物减少后，整体压力不明显下降（机制怪补回挑战）。
- [ ] Boss 战中 Boss 不会被小怪和特效淹没。

### 5.7 地图与关卡变化

#### 需求描述

当前局内变化主要来自怪物数量和数值增长，玩家路线决策较少。本轮先加入低成本地图事件，提高局内变化。

#### 规则

- 第一批地图事件：
  - **危险区**：出现前有预警，进入后持续受到伤害。
  - **治疗点**：玩家靠近后恢复生命。
- 第二批再考虑资源点。
- 暂不做复杂障碍物、寻路地图或完整随机地图生成。
- 地图事件在游戏 `1.5-2` 分钟后开始出现（前期让玩家熟悉基本战斗）。
- 同屏最多 `1` 个危险区 + `1` 个治疗点，避免地图事件本身造成视觉混乱。
- **生成频率和位置规则详见 5.10**。

#### 危险区参数

| 属性 | 值 | 说明 |
|---|---|---|
| 预警时间 | `2` 秒 | 出现红色虚线圆圈闪烁 |
| 持续时间 | `6-8` 秒 | 激活后存在时间 |
| 半径 | `80-100` 像素 | 中等范围，不能过大 |
| 每次伤害 | 玩家最大血量的 `3-5%` | 持续伤害，停留越久越危险 |
| 伤害间隔 | `0.5` 秒 | 高频低伤，催促玩家离开 |
| 视觉 | 预警：红色虚线圆；激活：红色半透明区域 + 地面裂纹效果 | 需要与敌人攻击有区分度；z-index 见 5.9 |

#### 治疗点参数

| 属性 | 值 | 说明 |
|---|---|---|
| 治疗量 | 玩家最大血量的 `15-20%` | 一次性恢复，有拾取价值 |
| 存在时间 | `8-12` 秒 | 不会一直在，需要抓时机 |
| 触发半径 | `25-30` 像素 | 需要主动靠近 |
| 视觉 | 绿色光圈（视觉半径约 60px，远大于触发半径，便于发现）+ 十字图标 | 明确的治疗暗示 |
| 拾取后效果 | 绿色 +HP 飘字 + 光圈扩散消散 | 正向反馈 |

#### 验收标准

- [ ] 危险区出现前有明确的 2 秒预警动画。
- [ ] 玩家进入危险区会受到持续伤害，离开后伤害立刻停止。
- [ ] 治疗点触发后有明确的恢复反馈。
- [ ] 治疗点视觉半径大于触发半径，便于玩家发现。
- [ ] 地图事件不会遮挡主角、Boss 技能或经验球（z-index 严格按 5.9 规划）。
- [ ] 同屏地图事件数量不超过上限。
- [ ] 闯关模式 60s 关卡可能完全没有地图事件（预期，无需额外处理）；90-180s 关卡能见到 1-2 个。

### 5.8 模式适配（v0.5 新增）

#### 5.8.1 模式差异盘点

#### 5.8.2 各条款覆盖性

#### 5.8.3 分模式参数（方案 C）

**自由模式**：完全按原 PRD 时间触发（5min 后 -20%、5min 后机制怪 15% 概率混入）。

**闯关模式**：
- 不削减普通怪量（关卡短，本来就需要密度感）
- 机制怪从**第 2 章（关卡 6+）开始**渐进引入，按章节加权：

| 章节 | 关卡范围 | 机制怪出现概率 | 说明 |
|---|---|---|---|
| 第 1 章 | 1-5 | 0% | 让玩家熟悉基础战斗和 Boss 战 |
| 第 2 章 | 6-10 | 5% | 偶尔遇到，作为新机制引入 |
| 第 3 章 | 11-15 | 15% | 中等密度，玩家应已掌握应对 |
| 第 4 章 | 16-20 | 25% | 高密度，与精英怪共同构成压力 |

**实现方式**：在 `STAGE_DATA[stage_id]` 加新字段 `mech_enemy_chance: float`（如 `0.15`），spawner 在 `_spawn_normal_wave` 中按此概率把普通怪 spawn 替换为机制怪。

**机制怪类型选择**：每次按 50/50 在 `charger` 和 `ranged` 中随机。

#### 5.8.4 验收标准

- [ ] 闯关模式 1-1 关到 1-5 关都看不到机制怪。
- [ ] 闯关模式 2-1 关开始有概率出冲锋怪 / 远程怪。
- [ ] 闯关模式 4 章关卡机制怪密度明显高于 2 章。
- [ ] 自由模式 5min 后能看到机制怪（约 15% 比例）。
- [ ] Boss 战减少小怪在两种模式下都生效。
- [ ] 地图事件在闯关短关卡缺席不会被视为 bug。

### 5.9 z-index 规划与降噪执行清单（v0.5 新增）

#### 5.9.1 z-index 统一表

为了让 5.3 的视觉优先级有明确的代码实现依据，在 `game_data.gd` 中定义 z 常量统一管理：

| 常量名 | 值 | 用途 |
|---|---|---|
| `Z_BACKGROUND` | -10 | 背景瓦片（已有） |
| `Z_DANGER_ZONE` | -5 | 危险区（地面层） |
| `Z_HEALING_POINT` | -4 | 治疗点（地面层） |
| `Z_GEM` | 0 | 经验宝石（默认） |
| `Z_ENEMY` | 0 | 普通敌人（默认） |
| `Z_PLAYER_MARKER` | 0 | 主角脚底识别环（player `_draw` 内层） |
| `Z_PLAYER` | 1 | 主角主体（已有） |
| `Z_BOSS` | 2 | Boss（高于普通敌人） |
| `Z_TRAIL` | 5 | 武器拖尾（已有） |
| `Z_VFX_MID` | 7 | ring_wave / line_attack（已有） |
| `Z_VFX_HIGH` | 10 | hit_flash / spark_burst（已有） |
| `Z_FLOAT_TEXT` | 20 | 伤害数字（已有） |
| `Z_BANNER` | 50 | HUD 横幅 |
| `Z_SCREEN_FLASH` | 100 | 全屏闪光（已用 CanvasLayer.layer=100） |

新增节点（地图事件、机制怪标识符）使用上表常量；现有节点逐步替换硬编码值。

#### 5.9.2 降噪具体砍除清单

PRD v0.4 的"降低普通命中特效"过于模糊。具体执行：

| 项目 | 当前值 | 改为 | 适用范围 |
|---|---|---|---|
| `VfxPool.hit_flash` 普通敌人命中 size | 12 | 8 | 普通怪命中（不含 Boss、暴击） |
| `VfxPool.spark_burst` 普通命中 count | 6 | 3 | 普通武器命中（不含进化、Boss） |
| `weapon_base.spawn_damage_number` 偏移 | ±5 | ±3 | 所有武器 |
| `VfxPool.trail_attach` 普通武器 length | 8 | 4 | 普通武器（进化武器保持 8） |

**判定方式**：
- 是否暴击：`dmg >= 20`（沿用 `weapon_base.gd:79`）
- 是否进化武器：`weapon_type >= GameData.WeaponType.INFERNO_STORM`
- 是否 Boss 命中：检查 target 是否 in `bosses` group

暴击 / 进化武器命中 / Boss 命中**不在砍除范围内**，保留完整表现强度。

#### 5.9.3 验收标准

- [ ] 所有新代码使用 `GameData.Z_*` 常量，不硬编码 z_index 数字。
- [ ] 普通武器拖尾视觉明显短于进化武器。
- [ ] 普通怪命中的 hit_flash 不会喧宾夺主。
- [ ] Boss 命中、暴击、进化武器视觉仍保持原强度，玩家能感受到差异。

### 5.10 机制怪与地图事件接入规则（v0.5 新增）

#### 5.10.1 机制怪 ENEMY_TYPES 扩充

在 `game_data.gd` 的 `ENEMY_TYPES` 加 2 个新类型（参数详见 5.6）：

```gdscript
"charger": {
    "name": "冲锋者",
    "health": 50.0,
    "damage": 8,           # 普通追击伤害
    "charge_damage": 16,   # 冲刺伤害
    "speed": 24.0,         # 普通追击
    "charge_speed": 110.0, # 冲刺速度
    "xp_value": 8,
    "size": 14.0,
    "color": Color(0.85, 0.25, 0.15),  # 红色调
    "behavior": "charger",
},
"ranged": {
    "name": "巫医",
    "health": 24.0,
    "damage": 10,          # 弹幕伤害
    "speed": 30.0,
    "retreat_speed": 36.0, # 被追近时后撤速度
    "keep_distance": 140.0,# 保持距离
    "shoot_interval": 2.5, # 射击间隔
    "xp_value": 6,
    "size": 12.0,
    "color": Color(0.55, 0.3, 0.85),   # 紫色调
    "behavior": "ranged",
},
```

#### 5.10.2 sprite 复用规则（不新建美术资源）

- **冲锋怪**：复用 `zombie` sprite + 红色 modulate（暴怒感），头顶画一个**红色感叹号**（`_draw` 实现，3px 高度）
- **远程怪**：复用 `skeleton` sprite + 紫色 modulate（不祥感），头顶画一个**紫色弓箭符号**（`_draw` 实现）
- 头顶标识符的目的：让玩家在远距离也能区分机制怪和普通怪

#### 5.10.3 enemy_pool / spawner 接入

**自由模式**：在 `enemy_spawner.gd._spawn_normal_wave` 中：
```gdscript
if not GameData.is_stage_mode() and time >= 300.0:
    if randf() < 0.15:
        # 替换为机制怪
        type_key = ["charger", "ranged"].pick_random()
```

**闯关模式**：在 `STAGE_DATA[stage_id]` 加 `mech_enemy_chance` 字段：
- 第 1 章关卡（1-5）：不加字段（默认 0）
- 第 2 章关卡（6-10）：`mech_enemy_chance = 0.05`
- 第 3 章关卡（11-15）：`mech_enemy_chance = 0.15`
- 第 4 章关卡（16-20）：`mech_enemy_chance = 0.25`

spawner 读 `stage_data.get("mech_enemy_chance", 0.0)` 决定是否替换。

#### 5.10.4 机制怪行为实现要点

- **行为分支**：在 `enemy.gd` 的 `_physics_process` 中按 `behavior` 字段分发到不同分支
- **冲锋怪**：状态机 `IDLE → WARNING → CHARGING → COOLDOWN`，扑空后回 `IDLE`
- **远程怪**：每帧检查与玩家距离，<80px 后撤，>160px 接近，120-160px 停下射击
- **远程弹幕**：复用现有 `boss_projectile.gd`（已实现），新建紫色版本即可

#### 5.10.5 地图事件生成规则

**生成节点**：在 `game_manager.gd` 的 `_world_event_timer` 已有，扩展为分别管理：

```
危险区：每 20-30s 尝试生成一次（同屏已有则跳过）
治疗点：每 35-50s 尝试生成一次（同屏已有则跳过）
```

**生成位置**：
- 基准方向：玩家 `facing` 方向（最后移动方向）
- 距离：玩家前方 200-400px 随机
- 角度偏移：±60° 扇形随机
- 这样玩家有"跑过去拿 / 主动绕开"的决策；不会出现在玩家身后必然吃伤害

**首次生成**：游戏开始 1.5-2min 后激活两个 timer（按已有逻辑）。

#### 5.10.6 验收标准

- [ ] 冲锋怪和远程怪有清晰的视觉区分（颜色 + 头顶符号）。
- [ ] 冲锋怪状态机切换流畅，扑空后不卡死。
- [ ] 远程怪在玩家追近时正确后撤，距离稳定在 120-160px。
- [ ] 闯关模式按章节渐进引入机制怪，2 章稀有、4 章高频。
- [ ] 自由模式 5min 后机制怪占比约 15%。
- [ ] 地图事件生成在玩家前方扇形区域，不出现在身后。
- [ ] 危险区与治疗点能同时存在，但各自不超过 1 个。

## 8. 最终确认项

| 项目 | 当前结论 |
|---|---|
| 主角放大 | `1.25x` 起步，最多调到 `1.35x`，**只缩放子 Sprite2D** |
| 脚底识别环 | 常驻显示，半透明浅蓝色，**不跟随视觉缩放** |
| 碰撞区 | 不改变（player 本来就 `collision_layer=0`） |
| 被动视觉 | 10 个角色各有独特激活态视觉，紧贴身体小范围；默认画在 `player._draw()` |
| 鹰眼朝向 | 用 `player.facing`（最后移动方向） |
