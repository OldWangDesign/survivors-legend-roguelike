# 角色系统设计文档

## 概述

游戏共有 **10 个可选角色**，每个角色拥有不同的外观素材、基础属性、初始武器和独特被动技能，为玩家提供差异化的游戏体验。

部分角色需要通过游戏内成就解锁。

## 被动技能

### 1. 神秘幸存者 —— 求生本能

- **效果**：血量低于 30% 时，移动速度 +25%
- **设计理由**：幸存者的核心就是活下来，濒死时加速逃命，给玩家翻盘机会
- **触发条件**：实时检测，血量恢复后自动取消

### 2. 蓝甲骑士 —— 铁壁

- **效果**：受到所有伤害降低 20%
- **设计理由**：最直接的坦克被动，等于永久隐性护甲
- **触发条件**：被动永久生效

### 3. PICO-8 复古 —— 复古幸运

- **效果**：获得经验值 +20%
- **设计理由**：平衡型角色通过更快升级获得优势，更早解锁强力武器组合
- **触发条件**：拾取经验宝石时自动生效

### 4. 翠林游侠 —— 鹰眼

- **效果**：投射物武器射程和飞行速度 +25%
- **设计理由**：远程特化，站得更远打得更准，配合飞刀初始武器形成远程打法
- **触发条件**：被动永久生效，影响所有投射物类武器

### 5. 赤焰武士 —— 嗜血

- **效果**：每击杀 10 个敌人，获得攻击力 +15% buff，持续 5 秒，最多叠加 3 层（+45%）
- **设计理由**：鼓励积极近战砍杀，越杀越强形成正反馈循环
- **触发条件**：击杀计数器，满 10 时触发，计数器归零重新计算
- **满层效果**：1.3（基础）x 1.45（3层嗜血）= 1.885 倍攻击力

### 6. 暗夜法师 —— 魔力涌动

- **效果**：每 25 秒，所有当前持有的武器同时额外发射一次
- **设计理由**：周期性爆发，配合极低冷却（0.6）形成技能风暴
- **触发条件**：25 秒定时器，自动触发

### 7. Sweetie 柔和 —— 生命之泉

- **效果**：每 5 秒回复 1 点 HP
- **设计理由**：持续回血配合高血量（130），打持久战；弥补低攻击力的短板
- **触发条件**：5 秒定时器，不超过最大血量

### 8. 赛博朋克 —— 超频芯片

- **效果**：连续移动超过 2 秒后，冷却速度额外 +40%
- **设计理由**：跑打流玩法，停下来就失去加成，鼓励不停移动的高操作风格
- **触发条件**：检测持续移动时间，停止移动后 buff 立即消失
- **满状态效果**：0.6（基础冷却）x 0.6（超频）= 0.36 倍冷却，技能近乎连发

### 9. 暗黑哥特 —— 灵魂汲取

- **效果**：击杀敌人回复 1 点 HP
- **设计理由**：玻璃炮（60 血）的续命手段，杀得越多活得越久，形成"不进则死"的紧张玩法
- **触发条件**：每次击杀敌人时触发，不超过最大血量

### 10. 公主婚礼 —— 神圣庇护

- **效果**：受到致命伤害时免死，回复 30% 最大血量，冷却时间 60 秒
- **设计理由**：全通关隐藏角色的专属复活机制，强大但有长冷却限制
- **触发条件**：血量将降至 0 时自动触发，冷却期内不可再次触发

## 玩法策略分析

### 角色搭配化学反应

| 角色 | 核心策略 | 推荐武器搭配 |

| 暗夜法师 | 技能覆盖，周期爆发清屏 | 魔杖 + 闪电链 + 陨石 |
| Sweetie 柔和 | 跑图吸经验，靠等级碾压 | 圣水 + 吸血光环 + 护盾 |
| 赛博朋克 | 持续移动，跑打结合 | 旋风斩 + 飞刀 + 火球术 |

## 角色选择界面

### 布局设计（居中卡片式）

采用居中卡片式布局，所有信息集中在一张固定宽度（520px）的卡片中，角色精灵放大居中作为视觉焦点。

```
│               = 选择角色 =                         │
│                                                    │
│        ┌────────────────────────────┐              │
│        │  ◀   神秘幸存者「默认」   ▶  │              │
│        │                            │              │
│        │      ┌──────────┐          │              │
│        │      │          │          │              │
│        │      │ 角色精灵  │          │              │
│        │      │ 128x128  │          │              │
│        │      │          │          │              │
│        │      └──────────┘          │              │
│        │                            │              │
│        │   ⚔ 鞭子  |  ★ 求生本能   │              │
│        │    血量低于30%时移速+25%     │              │
│        │  ──────────────────────    │              │
│        │  血量  ██████░░░░░  100    │              │
│        │  速度  ████████░░░  200    │              │
│        │  攻击  ██████░░░░░  x1.00  │              │
│        │  冷却  ██████░░░░░  x1.00  │              │
│        │  拾取  ██████░░░░░  x1.0   │              │
│        │                            │              │
│        │  🔒 解锁条件: 通关第 X 关   │  (未解锁)     │
│        │  ✦ 已解锁 ✦                │  (已解锁)     │
│        └────────────────────────────┘              │
│                                                    │
│          [返回]    3 / 10    [确认选择]              │
└──────────────────────────────────────────────────┘
```

### 布局结构

| 区域 | 内容 | 说明 |
|------|------|------|
| 标题行 | `= 选择角色 =` | 金色大字，居中显示 |
| 卡片第1行 | `◀ 角色名「定位」 ▶` | 左右箭头 + 角色名和定位标签，居中对齐 |
| 卡片第2行 | 角色精灵预览 | 128x128 像素精灵，带暗色内框，居中展示，2帧动画 |
| 卡片第3行 | `⚔ 武器名 | ★ 被动名` | 初始武器和被动技能名称并排，紧凑布局 |
| 卡片第4行 | 被动技能描述 | 小号暗灰色字，居中显示 |
| 分割线 | `────────` | 视觉分组 |
| 卡片第5行 | 5 条属性条 | 血量/速度/攻击/冷却/拾取，带颜色和数值 |
| 卡片底部 | 状态标签 | 已解锁显示绿色 "✦ 已解锁 ✦"，未解锁显示红色解锁条件；始终占位保证卡片高度一致 |
| 页面底部 | `[返回] 3/10 [确认选择]` | 操作按钮 + 角色序号计数器 |

### 交互方式

- **鼠标操作**：点击 ◀ / ▶ 按钮切换角色
- **键盘操作**：← / → 切换角色，Enter 确认选择，Escape 返回
- **锁定状态**：未解锁角色显示 "LOCKED" 覆盖层 + 红色解锁条件文字，确认按钮不可点击
- **已解锁状态**：显示绿色 "✦ 已解锁 ✦" 提示，卡片高度与未解锁一致

- 血量：红色
- 移动速度：蓝色
- 攻击倍率：橙色
- 冷却倍率：绿色（越短越好，条越长表示冷却越快）
- 拾取倍率：紫色

> 注：冷却倍率在展示时做反转处理，0.6 显示为长条（冷却快 = 好），1.1 显示为短条（冷却慢 = 差）

## 技术实现要点

### 数据结构（game_data.gd）

```gdscript
const CHARACTER_DATA: Dictionary = {
    "mystery_survivor": {
        "name": "神秘幸存者",
        "style": "style_i_mystery",
        "role": "默认",
        "weapon": WeaponType.WHIP,
        "max_health": 100,
        "base_speed": 200.0,
        "damage_mult": 1.0,
        "cooldown_mult": 1.0,
        "pickup_mult": 1.0,
        "passive": "survival_instinct",
        "passive_name": "求生本能",
        "passive_desc": "血量低于30%时，移动速度+25%",
        "unlock_type": "free",
        "unlock_value": 0,
    },
    # ... 其余角色 ...
}
```

### 需要修改的文件

1. `scripts/game_data.gd` — CHARACTER_DATA、selected_character、解锁状态持久化
2. `scripts/ui/char_select.gd` — 新建角色选择 UI（精灵预览 + 属性条 + 解锁状态）
3. `scenes/char_select.tscn` — 新建对应场景
4. `scripts/player.gd` — _ready() 读取角色数据应用属性和初始武器、被动技能逻辑
5. `scripts/ui/main_menu.gd` — 自由模式跳转改为角色选择页
6. `scripts/ui/stage_select.gd` — 选关后跳转角色选择页
7. `scripts/game_manager.gd` — 结算时检查解锁条件
8. `scripts/ui/hud.gd` — HUD 顶部迷你角色指示器
9. `scripts/ui/stats_panel.gd` — 左侧面板角色完整信息

## 游戏内 HUD 设计

### 整体布局

采用三行式布局，信息分层清晰：

```
Row 1: [经验条 EXP 21/40 ══════════════════════════════ Lv.3]  ← 全宽

Row 2: [头像][角色名 ]     01:41        [2-3 鬼影重重   ]
       [    ][★被动状态]                [击杀: 19/60    ]
                                        [═══════════    ]

Row 3: [武器1][武器2][武器3]
```

### 角色头像系统

| 特性 | 说明 |
|------|------|
| 尺寸 | 36x36 像素，带边框面板 |
| 来源 | 直接使用角色精灵第一帧 |
| 呼吸效果 | 持续微弱缩放脉动（±2%），让头像有"活"的感觉 |
| 受伤闪红 | 检测到 HP 下降时，头像边框和精灵闪红色 0.3 秒 |
| 被动闪色 | 被动技能从未触发变为触发时，头像边框闪绿色 0.3 秒 |

### 信息区域

| 位置 | 内容 | 说明 |
|------|------|------|
| 左侧 | 角色头像 + 角色名 + 被动状态 | 整合为一个信息块，始终可见 |
| 居中 | 时间 | 大号金色字体，26px |
| 右侧 | 关卡名 + 目标文本 + 目标进度条 + 击杀数 | 关卡模式显示完整信息，自由模式只显示击杀数 |
| 底行 | 武器栏 | 左对齐，动态显示已获得的武器 |

### 左侧 Stats Panel

按 TAB 展开的详细面板中，在属性列表上方显示完整角色信息（角色名、定位、被动名称、实时状态、被动描述）。
