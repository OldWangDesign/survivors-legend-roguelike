# 主菜单背景设计文档

## 概述

主菜单背景采用**自动战斗模拟**方案，展示不同角色使用各种武器技能与敌人战斗的动态场景。每次打开菜单都能看到不同的角色组合，增强视觉吸引力和游戏代入感。

---

## 设计方案

### 核心机制

| 特性 | 说明 |
|------|------|
| 角色轮换 | 每 9 秒自动切换到下一个角色，带淡入淡出过渡 |
| AI 移动 | 角色自动向最近敌人移动，保持安全距离，有走位逻辑 |
| 自动攻击 | 根据角色初始武器类型发射对应的武器特效 |
| 敌人生成 | 持续从屏幕四周涌入，被击中后死亡产生粒子效果 |
| 角色精灵 | 使用实际游戏内的角色精灵贴图，4x 放大，带行走动画 |
| 敌人精灵 | 使用实际游戏内的敌人精灵贴图，受伤时闪红 |

### 武器特效类型

| 武器 | 特效样式 |
|------|----------|
| 鞭子 | 弧形扫击线，快速消失 |
| 火球术/魔杖/飞刀 | 投射物（带尾迹），飞向最近敌人 |
| 圣经/十字架 | 环绕角色旋转的物体 |
| 圣水/大蒜/闪电 | 以角色为中心的 AOE 圆形扩散 |

### 角色轮换逻辑

1. 游戏启动时随机选择一个角色
2. 缓存所有 10 个角色的精灵帧（避免运行时切换）
3. 每 9 秒触发切换：
   - 0.3 秒淡出（屏幕变暗）
   - 中间点切换角色和武器
   - 0.3 秒淡入（屏幕恢复）
4. 底部显示当前展示角色的名称（金色小字）

### AI 行为

- **追踪**：距离最近敌人 >80px 时向其移动
- **后退**：距离 <50px 时后退保持安全距离
- **走位**：距离在 50-80px 时横向移动（垂直于敌人方向）
- **边界限制**：不超出屏幕边缘 60px

### 技术实现

- 基于 `Control._draw()` 绘制，使用 `draw_texture_rect()` 渲染角色和敌人精灵
- 启动时通过 `GameData.switch_style()` 循环加载所有角色精灵并缓存
- 敌人精灵使用当前已加载的敌人贴图
- 不依赖实际游戏系统（Player、Weapon、Enemy 节点），纯视觉模拟
- 主菜单 UI 叠加在背景之上，背景加暗角遮罩增加层次
